# 作品集網站 — 新手上線指南

給 Joel 的手把手教學。目標：把 `index.html` 這個作品集網站，免費放到網路上讓別人看得到。全程不需要寫程式，照著做就行。

---

## 一、先搞懂：整個網站是怎麼「架起來」的

做一個網站，其實只有三個角色在合作。用蓋房子來比喻：

| 角色 | 工具 | 負責什麼 | 比喻 |
|------|------|----------|------|
| **設計** | Figma | 決定長什麼樣子：排版、配色、字級、間距 | 建築藍圖 |
| **程式碼** | HTML / CSS / JS（就是 `index.html`）| 把設計變成瀏覽器看得懂的「真正的網頁」 | 實際蓋出來的房子 |
| **上線** | GitHub Pages | 把房子放到一塊「大家都到得了的地」，給你一個網址 | 土地＋門牌 |

**關鍵觀念**：Figma 畫得再漂亮，它本身還不是「網站」，只是圖。中間一定要有人把設計「翻譯」成 `index.html` 這種程式碼，網站才會真的能點、能捲動、能互動。**這段翻譯，就是我可以直接幫你做的部分。**

## 二、你的路線選擇（你要 Figma + GitHub，我建議走這條）

做設計師作品集，常見有三條路，各有取捨：

**路線 A｜Figma Sites 直接發佈（最快，但不走 GitHub）**
Figma 現在有「Figma Sites」功能，可以在 Figma 裡排好版，按一個鍵就發佈成網站。缺點：它是發佈到 Figma 自己的空間（可綁自訂網域），**跟 GitHub 沒關係**。適合完全不想碰程式的人。

**路線 B｜Figma 設計 → 轉成程式碼 → 放上 GitHub Pages（← 建議你走這條）**
你在 Figma 排版設計，我幫你把設計轉成 `index.html`，再放到 GitHub Pages 上線。
優點：完全免費、你完全掌控、履歷上「有一個自己部署的網站」對設計師是加分項，而且想改什麼我都能幫你改。**這正是你說的「Figma 排版 + GitHub 部署」，也是這份指南教的路。**

**路線 C｜用框架（如 Next.js）**
未來若要做到「每個作品有獨立內頁、可後台管理」再考慮。現階段對你來說太重，先不用。

> 你現在手上的 `index.html` 就是路線 B 的成果雛形。下面先教你怎麼上線，再教你怎麼改內容。

## 三、把網站放上網路：GitHub Pages 逐步教學

跟著做，大約 10 分鐘。全程在瀏覽器裡點一點就好。

**Step 1　申請 GitHub 帳號**
到 [github.com](https://github.com) → 右上角 Sign up → 用 Email 註冊（免費）。

**Step 2　建立一個「倉庫」（Repository，放檔案的地方）**
1. 登入後點右上角 **`+`** → **New repository**。
2. Repository name 填一個名字，例如 `portfolio`。
3. 選 **Public**（公開，才能免費用 Pages）。
4. 按 **Create repository**。

**Step 3　上傳你的 `index.html`**
1. 進到剛建好的倉庫頁面，點 **Add file ▾** → **Upload files**。
2. 把 `index.html` 拖進去（**檔名一定要是 `index.html`，且放在最外層根目錄**，這樣 GitHub 才會自動當成首頁）。
3. 下面按 **Commit changes**（等於「存檔」）。

**Step 4　打開 GitHub Pages 開關**
1. 在倉庫頁面點上方 **Settings**（齒輪）。
2. 左側選單找到 **Pages**。
3. 「Build and deployment」→ Source 選 **Deploy from a branch**。
4. Branch 選 **`main`**、資料夾選 **`/ (root)`** → 按 **Save**。

**Step 5　拿到你的網址**
等 1～2 分鐘，重新整理 Pages 頁面，最上面會出現網址，格式是：

```
https://你的帳號.github.io/portfolio/
```

點進去，你的作品集就上線了 🎉。以後只要重新上傳修改過的 `index.html`，網站會自動更新。

## 四、怎麼修改網站內容（不用會寫程式）

用「記事本」或免費編輯器 [VS Code](https://code.visualstudio.com/) 打開 `index.html`，我在檔案裡都用中文註解標好了可以改的地方。找 `←` 這個符號就對了。

**改文字**：直接找到畫面上的英文字（例如 `Defense`、你的標語），改成你要的字即可。

**換主題顏色**：檔案最上面第 `1` 區 `:root` 裡，把 `--accent: #00A3FF;` 換成你喜歡的色碼（例如橘色 `#ff6a3d`），整站點綴色會一起變。

**換作品圖片**：每張作品卡片裡有一行 `<img ... src="網址">`。目前用的是你舊網站的圖。要換成自己的圖，有兩個做法：
- 簡單版：把圖也一起上傳到 GitHub 倉庫，然後把 `src` 改成檔名，例如 `src="defense.jpg"`。
- 進階版：日後我可以幫你把圖整理成資料夾結構。

**新增／減少作品**：一整組 `<a class="card">…</a>` 就是一張卡片，複製一整組貼上就多一張，刪掉就少一張。

> 改完存檔 → 回到 GitHub 用 **Upload files** 重新上傳同名檔案覆蓋 → 網站幾秒後自動更新。

## 五、之後的固定工作流程（Figma ↔ 網站）

1. 在 **Figma** 裡排你想要的版面 / 新作品頁。
2. 把 Figma 畫面或連結給我，我幫你把它**轉成對應的 `index.html` 程式碼**（含互動效果）。
3. 你把更新後的檔案**上傳到 GitHub** → 網站自動更新。

你負責「設計決策」，我負責「翻譯成程式碼」，GitHub 負責「上線」。這樣分工你會很輕鬆。
