## 二、資料夾結構

```
Web/
├─ index.html                ← 首頁 A 版（已上線 GitHub Pages 的版本）
├─ index-b.html              ← 首頁 B 版（高階視覺重製版，供比對）
├─ defense.html … experimental.html   ← 6 個分類頁
├─ moai-router.html … （22 個作品內頁）
├─ about.html / contact.html
├─ assets/
│  ├─ site.css               ← 內頁共用樣式（改主題色改這裡的 --accent）
│  └─ images/                ← 437 張舊站原圖，依 Wix 分頁分資料夾
│     ├─ moai-router/ … （每個作品頁一個資料夾）
│     ├─ defense/ … （每個分類頁一個資料夾）
│     ├─ home/ about/         ← 首頁、About 專用圖
│     └─ shared/              ← 全站共用（logo、社群 icon）
│     ※ 一張圖只存一份：同時出現在多頁的圖，歸屬到「最專屬」的頁面
│       （作品頁 > 分類頁 > About/Contact > 首頁）。
│       完整對照表：content/image_locations.tsv
├─ 文案/                     ← 31 頁的文案 txt（含【圖 N】位置標記），要改文字先改這裡
│                              改完請 Claude「把文案同步進網頁」即可更新 HTML
└─ content/                  ← 純資料備份（不影響網站運作，勿刪）
   ├─ text/<頁名>.txt         ← 每頁抽取出的完整文案
   ├─ sequence/<頁名>.seq     ← 每頁「文字+圖片」的原始排列順序
   ├─ page_images.tsv         ← 頁面 ↔ 圖片對照表
   ├─ shared_images.txt       ← 全站共用圖（logo、社群 icon）
   └─ missing_images.txt      ← 未能下載的 7 張圖
```

## 七、之後上線 GitHub Pages 的注意事項

1. 圖片共 297MB——GitHub 單一 repo 建議 < 1GB，可以放，但**建議先壓縮**：
   之後可以請 Claude 把 `assets/original/` 轉成網頁最佳化版本（WebP、壓到 ~30–50MB），
   原圖留在電腦當母檔備份。
2. 上傳整個資料夾（html 檔 + `assets/`；`content/` 可以不上傳，它只是備份資料）。
3. 舊站網域 `joel-chang-design.com` 目前仍指向 Wix，**尚未受任何影響**。
